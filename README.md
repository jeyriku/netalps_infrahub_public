# NetAlps Infrahub Demo

Network CI/CD demo with **Infrahub as SSOT** (Single Source of Truth) for automated FRR configuration deployment.

Extension of `netalps_demo` with:
- **4 FRR routers** instead of 2, in chain topology
- **Infrahub** as central source of truth for all network parameters
- Dynamic FRR config generation from the Infrahub API
- SSOT audit in PyATS tests (deployed vs Infrahub consistency)

## Infrahub SSOT Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                      Infrahub (SSOT)                        │
│  http://localhost:8000                                    │
│                                                             │
│  Schema: NetalpsNetworkDevice + NetalpsInterface             │
│  Data: devices, IPs, OSPF area/passive, BFD params           │
└──────────────────────────┬──────────────────────────────────┘
                           │ API GraphQL
                           │
          ┌────────────────┼────────────────┐
          │                │                │
   load_data.py   generate_configs.py    post_check.py
   (PUSH data)   (PULL → frr.conf)      (audit SSOT)
```

**CI/CD Flow:**
```
[infrahub_start] → [load_schema] → [load_data] → [generate_configs]
      → [deploy_lab] → [pre_check] → [post_check] → [cleanup]
```

## Quick Start

### 1. Prerequisites

```bash
# Required Docker images
docker pull frrouting/frr:latest
docker pull nicolaka/netshoot:latest

# Venv with dependencies
source .venv/bin/activate
```

### 2. Start Infrahub

```bash
cd netalps_infrahub/infrahub
docker compose up -d

# Wait for Infrahub to be ready (~2 min)
bash ../scripts/wait_for_infrahub.sh
```

### 3. Load schema and data

```bash
cd netalps_infrahub

# Load the schema
INFRAHUB_API_TOKEN=satoken infrahubctl schema load infrahub/schema/network.yml

# Load network data
INFRAHUB_TOKEN=satoken python infrahub/load_data.py

# Optional: force an idempotent refresh of OSPF/BFD data on existing objects
INFRAHUB_TOKEN=satoken python infrahub/load_data.py
```

### OSPF data in Infrahub (SSOT)

All OSPF settings are stored in `NetalpsInterface` and used to generate FRR config:

- `ospf_enabled`
- `ospf_passive`
- `ospf_area`
- `ospf_network_type`
- `bfd_enabled` and BFD timers

Quick check example:

```bash
# Read current OSPF area from SSOT
python scripts/set_ospf_area.py --device frr-rtr-03 --interface eth1
```

### 4. Generate FRR configs from Infrahub

```bash
# Generates configs/frr-rtr-0X/frr.conf from the Infrahub API
INFRAHUB_TOKEN=satoken python scripts/generate_configs.py

# Preview without writing
python scripts/generate_configs.py --dry-run
```

### 5. Deploy the lab and validate

```bash
# Deploy the Containerlab lab
containerlab deploy -t topology.clab.yml

# Wait for OSPF convergence (30s)
sleep 30

# PyATS tests
python tests/pre_check.py --testbed tests/testbed.yml
python tests/post_check.py --testbed tests/testbed.yml

# Or via the full PyATS job
pyats run job tests/test_job.py --testbed-file tests/testbed.yml
```

### 6. Cleanup

```bash
containerlab destroy -t topology.clab.yml --cleanup
cd infrahub && docker compose down -v
```

## SSOT Failure Demo (OSPF area mismatch)

This demo injects a failure by changing OSPF area in Infrahub, then proves
that validation fails, and finally restores the original SSOT data.

```bash
cd netalps_infrahub
source .venv/bin/activate
INFRAHUB_TOKEN=satoken bash scripts/failure_demo.sh
```

What happens automatically:

1. Baseline generation + deploy + pre/post checks (PASS)
2. SSOT mutation: `frr-rtr-03/eth1 ospf_area -> 1`
3. Regenerate FRR config + reload router
4. `post_check` fails (expected)
5. Restore original `ospf_area`
6. Regenerate + reload + `post_check` passes again

## Interface Infrahub

Web access: **http://localhost:8000**
- Login : `admin` / `infrahub`
- Token API : `satoken`

Available models:
- **NetalpsNetworkDevice**: network devices (routers + hosts)
- **NetalpsInterface**: interfaces with OSPF/BFD parameters

## Differences from netalps_demo

| | netalps_demo | netalps_infrahub |
|---|---|---|
| Routeurs FRR | 2 | 4 |
| SSOT | Non | Infrahub (GraphQL API) |
| Configs | Static | Generated from Infrahub |
| Tests SSOT | No | Infrahub audit in post_check |
| Stages CI/CD | 4 | 6 (+ infrahub + configure) |
| OSPF/BFD Neighbors | 1 per rtr | 1 (edge) or 2 (transit) |
